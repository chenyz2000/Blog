MyBatis

MyBatis是ORM框架

ORM，Object Relational Mapping，对象关系映射。三个单词分别指Java的对象，数据库的关系，和映射

ORM框架，基于Java的持久层框架，实现了数据持久化

连接Java程序和数据库，使Java开发者可以以面向对象的思想来管理数据库，支持通过配置或注解，将接口和POJO映射成数据库中的记录

是对JDBC的封装，相对于传统的JDBC，简化了连接、操作的过程，减少了一半的代码量，只需要把关注点放在SQL的增删改查上

除了MyBatis以外，还有Hibernate、Spring Data。MyBatis是半自动的，Hibernate是全自动的，Spring Data是基于注解的

jar包目录为org.apache.ibatis

MyBatis中SQL语句写在xml里，从代码中分离，实现了与代码的解耦合

对象和关系、属性和字段的映射是在xml里

Java和MyBatis框架的对接

SqlSession是连接对象

  1. SqlSessionFactoryBuilder通过Build()建造SqlSessionFactory工厂对象
  2. SqlSessionFactory调用openSession()获得一个SqlSession对象
  3. 从SqlSession中获取Mapper接口
  4. 调用Mapper的方法

开发方式

原生接口，mapper代理

过程

  1. 创建maven项目，添加mybatis和mysql-connector-java两个依赖

  2. 在数据库里建表

  3. 在Java里建立数据表对应的实体类，属性要和数据表的字段对应

  4. 配置MyBatis，全局配置文件config.xml放在resource中，文件名也可自定义

使用原生接口

  1. 为每个实体类创建对应的Mapper.xml，SQL语句写在其中，以AccountMapper.xml为例：

    除了insert标签，还有update、delete、select，四者同理

    namespace是当前mapper文件的全路径，不加后缀名

    id是实际调用MyBatis方法时传入的参数，用来标示sql语句的id

    parameterType是调用对应方法时，方法参数的类型

    方法参数只有一个时（基本数据类型、String、数据类型包装类），sql语句里#{name}的name随便写

    方法参数有多个时，省略parameterType，sql语句里用下标#{arg0}、#{arg1}取

    方法参数为一个Bean对象时，sql语句可以用#{name}使用参数对象的属性

    resultType是对应方法的返回值类型，用于select中

    若方法返回值为Bean，select查询结果的字段与Bean的属性对应赋值，对应不上的不赋值（若sql里有as则根据as后的字段名），有多个重复字段（多表联合查询会有这种可能）的赋第一个字段

  2. 在全局配置文件config.xml中注册AccountMapper.xml

    resource写Mapper.xml的路径，带后缀，所以不能用.得用/

    通过config.xml确定数据库和mapper.xml，通过mapper.xml确定方法和sql语句

  3. 加载config.xml，创建sqlSession

    • InputStream in = Resources.getResourceAsStream(config.xml);

  4. 调用MyBatis原生接口来执行添加操作

关于 “#{}” ： 在传统的 JDBC 的编程中，占位符用 “?” 来表示，然后再加载 SQL 之前按照 “?” 的位置设置参数。而 “#{}” 在 MyBatis 中也代表一种占位符，该符号接受输入参数，在大括号中编写参数名称来接受对应参数。当 “#{}” 接受简单类型时可以用 value 或者其他任意名称来获取。

关于 “${}” ： 在 SQL 配置中，有时候需要拼接 SQL 语句（例如模糊查询时），用 “#{}” 是无法达到目的的。在 MyBatis 中，“${}” 代表一个 “拼接符号” ，可以在原有 SQL 语句上拼接新的符合 SQL 语法的语句。使用 “${}” 拼接符号拼接 SQL ，会引起 SQL 注入，所以一般不建议使用 “${}”。
