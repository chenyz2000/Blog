导入的依赖

spring-boot-starter-web

其中Spring-boot-starter指Spring Boot场景启动器

Spring Boot将所有的功能场景都抽取出来，做成一个个==Starters==（启动器），只要在项目里引入启动器，相关场景的所有依赖都会导入进来

Starters包含多种，web是web模块正常运行所依赖的组件，此外还有amqp，data-redis等

主程序类

@SpringBootApplication注解表示SpringBoot的主配置类，SpringBoot应该运行这个类的main方法来启动SpringBoot应用。使用这个注解能实现自动扫描包和自动配置默认配置的功能。

它其实是许多注解的组合注解

  • @SpringBootConfiguration表示SpringBoot的配置类

    其基于Spring的配置类@SpringBootConfiguration

  • @EnableAutoConfiguration开启自动配置

    其包括@AutoConfigurationPackage和@Import({EnableAutoConfigurationImportSelector.class})

    • @AutoConfigurationPackage，自动配置包

      • 包括@Import({Registrar.class})，Import是Spring的底层注解，给容器中导入一个组件。Registrar类的代码为

    • @Import({EnableAutoConfigurationImportSelector.class})，给容器中导入组件，组件为导入组件的选择器

      EnableAutoConfigurationImportSelector的父类AutoConfigurationImportSelector有方法public String[] selectImports(AnnotationMetadata annotationMetadata)，将所有需要导入的组件以全类名的方式返回，这些组件就会被添加到容器中，会给容器中导入非常多的自动配置类(xxxAutoConfiguration)，就是给容器中导入这个场景所需要的所有组件，并配置好这些组件，有了自动配置类，免去了手动编写配置、注入功能组件等工作

遇到问题

  • jdbcTemplate为null。将测试程序和main所在的包保持统一，并注意写完整测试程序的注解
  • 连接mysql出现caching_sha2_password错误。默认的mysql-connector-java版本是5.7左右，连接8.0版本的mysql加密方式会出错，在pom.xml中将mysql-connector-java改为8.0以后的版本即可
  • 连接mysql出现timezone错误。在jdbc的url后加上?serverTimezone=UTC
