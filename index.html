MVC与三层架构没有关系，它们之间并不对应

MVC架构模式

MVC并没有明确的定义，不同的实现有不同的细节

  • Model：模型，包含核心功能和数据（数据+业务逻辑）
  • View：视图，代表用户界面（展示数据）
  • Controller：控制器作用于模型和视图上。它控制数据流向模型对象，并在数据变化时更新视图。它使视图与模型分离开；接受用户的输入并调用模型和视图去完成用户的需求（应用程序逻辑）

Vue

Posted on 2020-07-23 | Post modified: 2020-08-18 | In Front End

Vue

前端框架，擅长于数据操作、表单填写、页面内容根据数据进行渲染、页面元素部分刷新

页面由数据生成，只要数据就可以进行页面视图的渲染

前端发展过程：

  1. 原生JS
  2. jQuery等类库。解决了浏览器大部分兼容性问题
  3. 前端模板引擎。调用模板引擎的方法生成DOM元素渲染页面，而不是直接操作DOM，解决了频繁操作DOM的问题
  4. Vue等前端框架。只渲染变更内容，减少不必要的操作，提高渲染效率，解决渲染整个页面的问题

框架是一套完整的技术解决方案，项目选择框架后难以更改；类库提供的是单一的小功能，对项目的侵入性较小

MyBatis

Posted on 2020-08-05 | Post modified: 2020-08-13 | In Spring

MyBatis

MyBatis是ORM框架

ORM，Object Relational Mapping，对象关系映射。三个单词分别指Java的对象，数据库的关系，和映射

MyBatis是ORM框架，是基于Java的持久层框架，实现了数据持久化

连接Java程序和数据库，使Java开发者可以用面向对象的思想来管理数据库，支持通过配置或注解，将接口和POJO映射成数据库中的记录

是对JDBC的封装，相对于传统的JDBC，简化了连接、操作的过程，减少了一半的代码量，只需要把关注点放在SQL的增删改查上

除了MyBatis以外，还有Hibernate、Spring Data。MyBatis是半自动的，Hibernate是全自动的，Spring Data是基于注解的

Ajax

Posted on 2020-08-10 | Post modified: 2020-08-10 | In Front End

网络请求

通过Ajax、Fetch、WebSocket等API实现

Fetch

替代了Ajax

API是基于JavaScript Promise设计的

fetch是全局量window的一个方法

Ajax

Asynchronous JavaScript and XML，异步的JS和XML

同步：发完请求进行等待，获得响应后才会执行之后的代码

异步：发完请求不管响应，但会监听某个状态，当状态发生改变达到要求时，再作出相应反应

无需重新加载整个网页就可以向服务器发送请求，网页的局部数据刷新

适用场景：注册时查询用户名是否已存在，登录时提示用户名密码错误

Ajax的核心是XMLHttpRequest对象（XHR），XHR为向服务器发送请求、解析服务器响应提供了接口，使浏览器可以发出HTTP请求与接收HTTP响应、实现在页面不刷新的情况下和服务端进行数据交互

RabbitMQ

Posted on 2020-08-09 | Post modified: 2020-08-09 | In Spring

消息队列

消息队列Message Queue

页面发布到服务器上，页面发布程序直接和每一个服务器的程序通信，叫做同步操作，同步操作是一步做完了才能做下一步

服务器的程序监听一个消息队列MQ，页面发布程序把消息发布给MQ，MQ把消息通知给监听它的每一个程序，MQ使发布程序和接收程序间异步处理，并且解耦合

Redis

Posted on 2020-08-07 | Post modified: 2020-08-07 | In Spring

Redis

非关系型数据库，缓存数据库，有点类似于mongodb不过是基于内存进行存储

数据存储在内存中而不会存在硬盘的文件中，结构是K-V对，没有数据表的概念，只有一个整体的字典

持久化存储，即使Redis服务关闭了，数据也在内存中

底层使用C语言编写，Java可以通过Spring Boot调用Redis

支持数据类型

字符串、列表、集合、有序集合、哈希

除了哈希其他的存储形式都是{key:value}，哈希是{key:{key:value}}即通过key找到一个map
