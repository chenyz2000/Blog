yum更新163源

  • cd /etc/yum.repos.d/

  • 按照自己的版本下载源，CentOS 7，使用命令：

    wget http://mirrors.163.com/.help/CentOS7-Base-163.repo

  • mv CentOS7-Base-163.repo CentOS-Base.repo

  • 运行以下命令生成缓存：

    yum clean all

    yum makecache


开启SSH访问

修改/etc/ssh/sshd_config文件

将文件中“PasswordAuthentication”的值修改为“yes”

若想允许root远程登录，将PermitRootLogin值改为yes

之后重启sshd服务service sshd restart

SSH访问ECS不仅需要修改sshd服务，还要在安全组中设置放行

挂载硬盘

fdisk -l查询硬盘的dev名，需要将其挂载到/mnt/下

默认的挂载命令是mount /dev/sdb2 /mnt/win

如果硬盘是ntfs格式，使用默认的挂载，只能读取不能写入

需要使用ntfs-3g进行挂载

Debian自带ntfs-3g，不用安装

用其进行挂载的命令为mount -t ntfs-3g /dev/sdb2 /mnt/win

理论上此时已经可以读写使用了

不过，在操作中出现了一个问题

使用ntfs-3g进行挂载仍然不能写入

这是因为此硬盘在windows下时，系统快速启动的文件存储在上面，强迫其他系统只能读取

我把这个硬盘放回windows电脑，关闭快速启动，重启电脑，自动删除文件

再拿回来挂载，问题解决了

自动进行挂载则可以修改/etc/fstab

在其中添加/dev/sdb2 /mnt/win ntfs-3g default 0 0

sz/rz传输文件

基于ZModem传输协议

首先安装lrzsz服务yum install lrzsz

从服务器传输文件到本机sz filename1 filename2

从本机传输文件到服务器rz，xshell可以直接拖动进去

任务前后台切换

  • 命令后加&，可以把这个命令放到后台执行

  • ctrl + z，可以将一个正在前台执行的命令放到后台并暂停

  • jobs，查看当前有多少在后台运行

  • fg %n，将后台中的命令调至前台继续运行

  • bg %n，继续执行一个在后台暂停的命令

  • kill %n，杀死后台任务

其中，n是通过jobs命令查到的命令序号
