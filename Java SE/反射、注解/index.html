反射、注解

Posted on 2020-05-30 | Post modified: 2020-05-30 | In Java SE

反射

java.lang.reflect包

功能

  • 运行中分析类的成员变量、方法

注解

源码=代码+注释+注解

注解就像是标签一样，不会对对象产生影响，只有用特殊的工具才可以访问注解

主要用途

  • 提供信息给编译器、IDE
  • 使用某些工具产生额外的代码、配置文件
  • 在程序运行时访问，提升程序的动态性

    类型

JDK预定义普通注解

直接修饰Java元素

  • @SuppressWarnings，压制警告
  • @Override，重写方法，提示IDE检查
  • @Deprecated，废弃
  • @SafeVarargs，对不定项参数只做安全操作
  • @FunctionInterface，声明功能性接口
JDK预定义元注解

用于修饰注解

  • @Target，设置注解目标范围
  • @Retention，设置注解保留范围
  • @Docomented，此注解可进入JavaDoc文档
  • @Inherited，此注解可继承
  • @Repeatable，此注解可重复应用
