Spring Boot

Posted on 2020-07-11 | Post modified: 2020-08-04 | In Spring

Spring Boot

官方文档

简化Spring，将其中复杂的XML配置Bean的部分砍掉，是Spring的再封装

核心思想：CoC约定大于配置，Spring Boot已经配置好了约定俗成的部分，开发人员只需要定制不符合约定的部分

Spring Boot使得易于集成，如集成数据库、缓存、权限、web容器、监控、消息队列

原本开发Web项目需要：配置maven依赖，配置web.xml加载Spring，配置数据库连接配置Spring事务，配置加载配置文件的读取开启注解，配置日志文件，…，部署Tomcat进行调试

而使用Spring Boot创建即可使用：建立一个Spring Boot工程，配置maven依赖，接可以直接启动

springBoot用于开发微服务中的一个个服务，springcloud用于管理springboot开发的多个服务

通过maven项目构建

  1. 创建maven项目

  2. 导入springboot相关依赖

    其中指定的版本就是Spring Boot的版本

Spring

Posted on 2020-07-16 | Post modified: 2020-07-19 | In Spring

Spring

Java EE领域的企业级开发框架，是软件设计层面的框架，能帮企业实现复杂的网站后端服务

分层

优势是可以将应用程序进行分层，开发者可以自主选择组件

MVC层：Structs2、Spring MVC

ORM层：Hibernate、MyBatis、Spring Data

两大核心机制

  • IoC控制反转：针对复杂的对象管理问题，将设计好的对象交给IoC容器而不是new。工厂模式。
  • AOP面向切面编程：代码解耦，不改变源代码功能流程的基础上加入新功能，扩展性。代理模式。

痛点：配置文件多

IoC控制反转

IoC创建对象，DI负责对象的依赖注入

传统程序开发中，对象是由调用者new创建出来的；但Spring框架中，对象由IoC容器创建，再推送给调用者，整个流程完成反转
