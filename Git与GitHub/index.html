Git与GitHub

Posted on 2020-07-13 | Post modified: 2020-07-13

版本控制

管理版本，随时回退到任何时刻

协同开发，自动合并代码

Git

git status查看当前仓库的状态信息，红色是未暂存，绿色是已暂存

git reset可以取消暂存

文件初始状态是untracked，add后成staged，commit后成unmodified，有提交记录的文件修改后成modified

只提交已暂存的文件

git log查看每个commit的哈希值、提交者、时间、信息

哈希值很长，但是只要前七位就能锁定

暂存的文件修改后，状态会变成红色

版本回退，git reset commitID，该版本之后的版本不保留在当前版本控制中

有参数–hard不保留所有变更，–soft保留变更且变更内容处于staged状态，–mixed保留变更且变更内容处于modified状态

不过之后的版本也能通过git reflog查到所有commit的七位commitID

分支

git checkout -b <name> <template>创建分支，name是分支名，template是模板，默认是当前分支

分支建立后，分支与原分支平行，不会相互影响

git branch查看所有分支

合并分支的变更

是变更，不是内容

git merge

git rebase变基，枚举变更的commit，continue依次变基

远程仓库

从本地推送到远程push

fetch从远程获知

fetch+merge相当于pull
