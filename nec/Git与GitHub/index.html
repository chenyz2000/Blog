Git与GitHub

Posted on 2020-07-13 | Post modified: 2020-07-13 | In nec.

版本控制

  • 管理版本，随时回退到任何时刻

  • 协同开发，自动合并代码

Git

参考1，参考2

文件四个状态

初始状态是untracked

add后成staged

commit后成unmodified

有提交记录的文件修改后成modified

基本快照

git status查看当前仓库的状态信息，红色是未暂存，绿色是已暂存

暂存的文件修改后，状态会变成红色

git diff能查看已写入暂存和又改动却未写入暂存的区别

git reset可以取消暂存

commit只提交已暂存的文件

提交历史

git log

–oneline参数显示简洁版本，–graph显示分支与合并

查看每个commit的哈希值、提交者、时间、信息

哈希值很长，但是只要前七位就能锁定

版本回退，git reset commitID，该版本之后的版本不保留在当前版本控制中

有参数–hard不保留所有变更，–soft保留变更且变更内容处于staged状态，–mixed保留变更且变更内容处于modified状态

不过之后的版本也能通过git reflog查到所有commit的七位commitID

分支管理

创建分支

git branch (name) 或 git checkout -b (name) [template]

差别是后者切换到新分支

template是模板，默认是当前分支

分支建立后，与原分支平行，不会相互影响

查看分支

git branch查看本地分支

-r参数查看远程分支

-a参数查看本地和远程分支

删除分支

git branch -d (name)

切换分支

git checkout (name)

合并分支

合并的是commit的变更，不是对两个内容进行合并

原有分支如图

git merge (name)

将另一分支的不同的commit合并到当前分支

保留被合并分支的记录

git rebase

变基，枚举变更的commit，continue依次变基

不保留被合并分支的记录

远程仓库

添加删除远程仓库

git remote add (name) (url)

git remote rm (name)

查看所有远程仓库

git remote

推送到远程仓库

git push (name) (branch)

获取远程仓库更新

git fetch从远程获知，之后merge远程分支到当前分支

或git pull，相当于fetch+merge

GitHub

Folk与Pull Request

与Clone不同，Folk将别人的仓库复制一份到自己的账号，并与原本的仓库有所联系

在自己Folk下来的仓库创建一个分支，在此分支上修改

之后可以将此分支PR给原仓库，等待对方的审核

若通过，对方将commit合并到原仓库；若不通过，则继续修改提交而不需要PR
