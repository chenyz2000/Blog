JSP

Posted on 2020-05-29 | Post modified: 2020-05-29 | In Java Web

动态网页相比静态的区别：随时间、地点、用户操作改变

JSP

JSP是服务器端（Tomcat服务器）脚本语言，动态网页需要使用

JSP就是在HTML中嵌套Java代码

HTML中写的内容直接显示在页面

而JSP翻译成Java，最后以class文件执行

index.jsp

基本格式

JSP执行流程

第一次访问

  1. 客户端提交请求
  2. tomcat服务端将JSP翻译成Java
  3. 编译成class文件

此处的Java文件准确来说是servlet文件

Jsp和servlet可以相互转化，JSP比较简略

早期没有JSP时，就用Java中的servlet类

第二次访问

  1. 直接访问class文件

  2. 若服务端代码修改，则按第一次访问进行

统一编码

  • jsp翻译到Java的编码：.jsp文件中的contentType和pageEconding属性
  • 浏览器读取时的编码：.jsp文件中的content属性
  • 文本文件的编码
    一般都用utf-8

Java运行Tomcat时，会复制一份配置文件

将web工程以包的形式上传到服务器，而不是部署到webapps中

JSP的页面元素

脚本Scriptlet

脚本转为HTML源代码body中的内容

  • <% 局部变量，常规Java语句 %>
  • <%! 全局变量、定义方法 %>，可以被其他段的语句调用
  • <%= 输出的字符串 %>，相当于<% out.print(字符串) %>

    多段脚本按照在整个JSP文件中的顺序进行，可以互相调用

在JSP脚本中println也不换行，换行需要<br>

可以解析字符串中的HTML代码

指令

  • <%@ page 属性：值 %>，其中属性有
    • language指定JSP页面编写的脚本语言，一般为”Java”
    • import导入的类
    • pageEncoding为JSP文件编码
    • contentType为浏览器解析编码
  • <%@ include %>
  • <%@ taglib %>

注释

  • HTML注释：<!-- -->，可以在源代码中看到
  • Java注释
  • JSP注释：<%-- --%>

一般而言，修改配置文件需要重启Tomcat，修改页面文件不需要重启

JSP九大内置对象

自带的，不需要new也能使用的对象

  • out：输出对象，向客户端（网页页面）输出（System.out是打印到控制台）

  • request：请求对象，客户端向服务端的请求

  • page, pageContext, request, response, session, application, config, exception

    request对象常见方法

  • String getParameter(String key)根据input标签的name属性，返回value属性的值

  • String[] getParameterValues(String key)返回数组

  • void setCharacterEncoding(“utf-8”)设置请求时的编码（Tomcat7以前默认为ISO8859-1，Tomcat8以后为utf-8）

  • getRequestDispacher(“跳转向的页面”).forward()请求转发

  • getServletContext()

request注册

register.jsp和show.jsp
